Инструкция по работе с сайтом

При наведении курсора на элемент интерфейса спустя 2 секунды по нему можно увидеть подсказку.
Для работы в навигационной панеле пользователям доступны следующие разделы:
  • СОРС - ссылка на главную страницу сайта
  • События - ссылка на основной раздел сайта для работы с рисковыми событиями
  • Админ - ссылка на раздел сайта для администрирования
  • Журнал - ссылка на раздел сайта с журналом входа пользователей
В правом верхнем углу навигационной панели отображается аккаунт текущего пользователя с кратким обозначением подразделения в скобках.
Для начала работы с сайтом следует перейти по ссылке "События".
Алгоритм работы с рисковыми событиями:
  1. Пользователь создаёт сообщение о рисковом событии и направлет его на проверку риск-координатору
  2. Риск-координтор проверяет сообщение о рисковом событии и направлет его на рассмотрение риск-менеджеру
  3. Риск-менеджер создаёт на основе сообщения о рисковом событии новое рисковое событие
  4. Риск-менеджер заполняет данные о подразделениях-владельцах рискового события
  5. Риск-координаторы подразделений-владельцев рискового события назначают ответственных сотрудников по проработке мероприятий
  6. Ответственные сотрудники создают план мероприятий по урегулированию рискового события
  7. Риск-координатор проверяет созданный план мероприятий
  8. Ответственные сотрудники создают отчёт о выполнении мероприятий по урегулированию рискового события
  9. Риск-координатор проверяет созданный отчёт о выполнении мероприятий
  10. Риск-менеджер закрывает рисковое событие после его урегулирования
Для обработки рисковых событий используются следующие статусы:
  • - draft (черновик) - сообщение со статусом draft доступно только его автору. При этом возможно производить его редактирование и удаление.
  • - check (на проверке) - статус check присваивается сообщению после его отправки риск-координатору на проверку. При этом пользователь больше не может производить его редактирование и удаление. Риск-координатор подразделения может внести в данное сообщение правки, отправить его обратно пользователю на доработку или отправить в Управление по рискам для создания рискового события.
  • - sign (на рассмотрении риск-менеджера) - статус sign присваивается сообщению после его отправки риск-координатором в Управление по рискам. При этом сообщение блокируется для редактирования. Риск-менеджер может создать на основе этого сообщения новое рисковое событие или включить его в уже созданное ранее рисковое событие, а также отправить его в подразделение на доработку
  • - refine (на доработке) - статус refine присваивается сообщению при отправке его доработку риск-координатором или риск-менеджером. При этом сообщение доступно для правки и удаления со стороны его автора и риск-координатора подразделения.
  • - open (в работе) - статус open присваивается сообщению после создания на его основе рискового события или включения в существующее рисковое событие.
  • - close (закрыто) - статус close присваивается сообщению после закрытия рискового события, в состав которого оно входило.
Элементы интерфейса:
  • - иконка означает наличие возможности получения детальной информации.
  • - иконка означает наличие возможности редактирования элемента после её нажатия.
  • - иконка означает возможность раскрытия элемента для получения дополнительной информации.
  • - иконка означает, что по элементу отображается дополнительая информация.
  • - иконка означает наличие возможности внесения новых данных после её нажатия.
  • отображается слева от имени пользователя в навигационной панели при получении ответа от сервера
